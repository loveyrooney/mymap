# 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요 (Project Overview)
**프로젝트명**: MyMap (가칭)
**목표**: 서울 및 수도권 지역의 대중교통 최적 경로를 안내하고, **위치 기반의 통합 실시간 도착 정보**를 제공하여 사용자의 이동 유연성을 극대화하는 서비스입니다.

## 2. 해결하고자 하는 문제 (Problem Statement)
*   **기존 서비스의 한계**: 네이버지도, 카카오맵 등 기존 서비스는 여러 경로 중 하나를 사용자가 "선택"해야만 해당 경로의 상세 이동 정보(도착 시간 등)를 보여줍니다.
*   **사용자 불편**: 환승 타이밍을 놓치거나 예기치 않은 지연 발생 시, 현재 위치에서 대안 경로를 찾기 위해 **다시 검색**하고 **다시 경로를 선택**해야 하는 번거로움이 있습니다.

## 3. 핵심 가치 제안 (Core Value Proposition)
**"유동적 경로 선택을 위한 클러스터링 기반 정보 제공"**

*   **최적 경로 자동 추천**: 소요 시간, 도보 거리 등을 종합적으로 고려하여 **최적 경로 2~3개**를 자동으로 선별합니다.
*   **통합 도착 정보 시각화**: 선별된 경로들에 포함된 **모든 교통수단(버스, 지하철, 따릉이)**을 현재 위치 기반으로 **클러스터링**하여 한 화면에 보여줍니다.
*   **실시간 유연성**: 사용자는 특정 경로 하나에 얽매이지 않고, 눈앞에 오는 버스나 지하철의 실시간 도착 정보를 보고 **즉각적으로 최선의 이동 방법을 선택**할 수 있습니다.

## 4. 주요 기능 (Key Features)

### 4.1 경로 탐색 및 추천
*   출발지/도착지 기반 대중교통 경로 탐색.
*   소요 시간, 환승 횟수, 도보 이동 거리를 가중치로 계산하여 상위 2~3개 최적 경로 필터링.

### 4.2 위치 기반 정보 클러스터링 (Clustering)
*   추천된 경로들 상에 존재하는 인접한 정류장/역사를 하나의 **클러스터**로 그룹화.
*   지도 상에 클러스터 단위로 마커를 표시하여 시인성 확보.

### 4.3 실시간 도착 정보 (Real-time Arrival)
*   선택된 경로 내 모든 교통수단의 실시간 도착 정보를 한 번에 조회.
*   **WebSocket**을 통해 사용자가 보고 있는 클러스터의 버스/지하철 도착 정보를 실시간으로 업데이트(Push).

### 4.4 사용자 이동 기록 (Journey)
*   사용자의 지난 이동 경로 및 즐겨찾는 경로 저장 및 관리.

## 5. 시스템 아키텍처 (System Architecture)

### 5.1 Tech Stack
*   **Client**: Android (Kotlin, Naver Maps SDK)
*   **Core Backend**: Java, Spring Boot (JPA, MySQL/H2) - 사용자 관리, 경로 알고리즘, 데이터 영속성 담당.
*   **Real-time Service**: Python, FastAPI (Asyncio, WebSocket) - 공공데이터(TOPIS, OpenAPI) 비동기 수집 및 클라이언트 Push 담당.

### 5.2 데이터 흐름 (Data Flow)
1.  **App**이 **Spring Boot** 서버에 경로 요청.
2.  **Spring Boot**는 최적 경로를 계산/선별하여 **App**에 반환 (정적 경로 데이터).
3.  **App**은 경로 상의 주요 지점(클러스터) 정보를 가지고 **FastAPI** 서버에 WebSocket 연결.
4.  **FastAPI**는 해당 지점들의 실시간 도착 정보(공공데이터 API)를 주기적으로 조회하여 **App**에 Push.

## 6. 데이터베이스 모델링 (Rough Draft)
*   **User**: 사용자 정보
*   **Journey**: 사용자의 이동 기록 (User 1:N Journey)
*   **Region/Cluster**: 지역 및 정류장 클러스터링 정보
*   **Transport(Bus/Subway/Bike)**: 교통수단 기초 데이터

## 7. 향후 확장 계획
*   따릉이(공공 자전거) 경로 연계 강화.
*   사용자 맞춤형 경로 추천 알고리즘 고도화 (선호 교통수단 반영 등).
